Project Overview

Primary Care-Family Link Application

The Thai Cleft Primary Care-Family Link Application is a comprehensive digital platform (Web & Mobile) developed to integrate the care network for patients with cleft lip, cleft palate, and craniofacial deformities across 8 provinces in Northern Thailand. Designed to overcome geographical barriers and transportation challenges for patients in remote areas, the system seamlessly connects specialized medical centers (SCFC), provincial hospitals, and Primary Care Units (PCU/Health Promoting Hospitals). This collaboration enables efficient case discovery, patient referrals, long-term follow-ups, and proactive home visits, ensuring a continuous and integrated care journey for both healthcare providers and patient families.

System Roles & Architecture

Comprehensive Roles

Tailored for every user

The Thai Cleft Primary Care-Family Link Application features six distinct user roles: 1) PCU/Health Center Staff, 2) Case Managers, 3) Hospital Personnel, 4) SCFC Officers, 5) Patients/Families, and 6) System Administrators (IT Admin). The system is developed across two primary platforms: Web and Mobile. The Mobile Application prioritizes a user-friendly and portable UI/UX design, tailored for frontline staff performing field visits and for patient families. Conversely, the Web Application is optimized for large-screen interfaces to facilitate comprehensive data dashboards and back-end management for SCFC officers and IT administrators.

Testing & Validation

Clickable Prototype

Iterative Development

The development of a comprehensive platform, including both Web and Mobile Applications, is designed to support the healthcare network and patients. During the process, a high-fidelity 'Clickable Prototype' or Demo was created to present to frontline users. The primary objective of this demo is to allow users to experience the actual interface and verify whether the workflow aligns with their operational practices. Throughout the testing phase, the team meticulously gathers feedback and additional requirements directly from users. This valuable insight is used to refine and finalize the design, ensuring the system is fully optimized and truly meets user needs before being handed over to the development team for the final programming stage.

An AI-Augmented Workflow for Managing and Designing a Highly Complex, Multi-Role Healthcare Coordination Platform.

Phase 01

Information Synthesis & Scope Alignment

Challenge

The project's Terms of Reference (TOR) and system flowcharts were highly complex, expansive, and involved intricate multi-role logic. Additionally, critical user feedback was scattered across numerous weekly stakeholder meeting recordings.

AI Solution

Established a Single Source of Truth by utilizing NotebookLM. I consolidated all raw data—ranging from the official TOR to audio recordings of client feedback sessions—into a centralized knowledge base.

Impact
  • Drastically reduced the time required for manual information parsing and complex data synthesis.
  • Eliminated feature leakage and misunderstanding by leveraging AI for instant, contextual QA regarding system requirements.
  • Enabled immediate deep-dive queries against cross-referenced AI resources to ensure absolute accuracy.

Phase 02

Agile Prototyping & Parallel Design Workflows

Challenge

Stakeholders required a granular, frame-by-frame UI review while simultaneously demanding interactive prototypes and system demos for user flow testing. Traditional linear workflows (Waterfall) would have created severe delivery bottlenecks.

AI Solution

Implemented an Agile Design Workflow by leveraging Figma AI to generate interactive prototypes directly from wireframes. This enabled rapid asset duplication and real-time syncing within the Figma design space as updates occurred.

Impact
  • Broke the traditional dependency where prototyping could only begin after final UI design approval.
  • Enabled parallel delivery of static screens and interactive demos, allowing for weekly client updates and faster feedback loops.
  • Accelerated the overall user testing phase by keeping design iterations synchronized with physical demo requirements.

Phase 03

Requirement Tracking & Cross-Platform Alignment

Challenge

Due to the platform's large scale and diverse user roles, client requirements shifted frequently during live discussions, occasionally diverging from the original TOR. This created a high risk of misalignment between the latest feedback, UI designs, and the codebase on GitHub.

AI Solution

Architected an Automated Quality Gate by creating a custom Gemini Gem. This AI agent bridged the gap between NotebookLM (containing the latest meeting summaries) and GitHub (the codebase). The Gem automatically cross-referenced the current code against the latest client requirements.

Impact
  • Enabled continuous alignment and rapid conflict detection (Discrepancy Detection) between shifting requirements, designs, and production code.
  • Ensured that the final implementation accurately fulfilled the stakeholders' evolving needs, despite ongoing changes.
  • Maintained absolute deployment integrity by keeping developer and designer pipelines continuously synchronized with live stakeholder inputs.
